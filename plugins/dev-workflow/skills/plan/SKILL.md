## 目的

計画者エージェント（@planner）が以下を一気通貫で実行する:

1. 仕様ヒアリング（grill-me）
2. 仕様書・実装計画書の作成（spec）
3. GitHub issue の作成（epic + task）
4. **ユーザーに確認を求めて停止**

## 実行

計画者エージェントに以下を依頼する:

```
@planner
「$ARGUMENTS」について計画フェーズを実行してください。

### Phase 1: 仕様ヒアリング
ユーザーに1つずつ質問し、仕様を確定させてください。
- 各質問に推奨案を提示する
- コードベースで解決できるものは調査して解決する
- すべての分岐が解決したら仕様サマリーを出力する

### Phase 2: ドキュメント作成
仕様サマリーをもとに以下を作成してください:
- docs/specs/$ARGUMENTS/spec.md（仕様書）
- docs/specs/$ARGUMENTS/plan.md（実装計画書）

### Phase 3: GitHub issue作成
実装計画書をもとに以下を作成してください:
- epicラベル付きの親issue（仕様書の内容）
- taskラベル付きの子issue（計画書の各タスク）
- 親issueのタスク一覧を子issue番号で更新

### Phase 4: 確認依頼
作成した成果物の一覧を表示し、ユーザーに確認を求めてください:
- 仕様書のパス
- 実装計画書のパス
- Epic issue番号と全Task issue一覧
- 「確認後 `/dev-workflow:run #[epic番号]` で自律実装を開始できます」と案内
```

## 注意

- 計画者はPhase 1のヒアリング中のみユーザーに質問する
- Phase 2以降は自律的に進める
- ユーザーの承認なしに実装は開始しない
